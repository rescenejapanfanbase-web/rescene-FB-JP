RESCENE JAPAN FANBASE
最強サイト化 第1段階：壊れない基盤（完了版）

【基本方針】
現在のデザイン、文章、画像、URL、Notion運用を変えず、内部構造と公開手順だけを強化しています。
公開サイトは引き続き静的HTMLとして配信されるため、共通部品の読み込み待ちによる表示崩れはありません。

【通常の更新】
GitHub Actionsの「Sync All Site Content」を使用します。
- 3時間ごとの5分に自動実行
- 手動実行にも対応
- 全同期を決められた順番で実行
- 全検査に成功した場合だけ1コミットで公開
- push競合時は最新mainから最大3回やり直し
- 内容に変更がなければコミットしない

【共通レイアウト】
生成元は以下です。
- templates/site-header.html
- templates/site-footer.html
- templates/site-header-contact.html
- templates/site-footer-contact.html

scripts/sync-site-shell.py が37ページへ静的に展開します。お問い合わせページは従来の専用ナビとCONTACTリンクを保つ専用テンプレートを使用します。
ルートページとarticles配下のリンク形式も自動で切り替えます。

従来33ページへコピーされていたメニュー・テーマ切替JavaScriptは削除し、
全37ページが js/common.js を1回だけ読み込む構成へ統一しました。

共通部分を変更するときは、テンプレートを修正後に以下を実行します。
python3 scripts/sync-site-shell.py

一致確認だけを行う場合：
python3 scripts/sync-site-shell.py --check

【画像表示領域の確保】
scripts/add-image-dimensions.py がローカル画像の実寸を取得し、
HTMLのwidth／heightを自動設定します。
CSS上の表示サイズは変更しません。

これにより、画像読込前後のレイアウト移動を抑えます。
以前38件あった画像寸法警告は0件になりました。

実行：
python3 scripts/add-image-dimensions.py

確認のみ：
python3 scripts/add-image-dimensions.py --check

SEO記事を再生成した場合も、統合同期内で画像寸法が再設定されます。

【公開前品質検査】
「Site Quality Gate」と scripts/run-prepublish-checks.sh が9段階で確認します。
1. Python構文
2. JavaScript／MJS構文
3. JSON
4. Workflow YAML
5. Workflow内Bash
6. 共通レイアウト一致
7. 画像width／height
8. HTML・CSS・JS・JSON・画像の内部参照
9. データ・HTML品質

追加検査：
- 重複IDとアンカー
- title、description、canonical
- ディスコグラフィのApple Music／Spotify
- メンバー、公式リンク、スケジュール、MV、YouTubeのデータ整合性
- JSONとフォールバックJSの一致

結果は data/quality-report.json で確認できます。
現在の基準値はエラー0件・警告0件です。

【公開前ビジュアル検査】
GitHub Actionsの「Build Site Preview」で画面幅ごとに独立実行します。
- 320px
- 390px
- 1024px
- 1440px

自動確認項目：
- 横スクロール
- JavaScriptページエラー
- トップページのカード件数
- PCナビ／ハンバーガーの切替
- ハンバーガーの開閉、ARIA、背景固定
- Languageがモバイルメニュー最下部にあること
- トップ公式リンク画像が56pxを超えないこと
- ライトモードで公式リンクの選択文字が見えること
- スマホ版ストリーミングに画面外要素がないこと

代表ページ19画面を描画し、スクリーンショットと統合レポートをArtifactへ保存します。

【緊急ロールバック】
GitHub Actionsの「Roll Back Site」を使用します。
1. 正常だったコミットSHAまたはタグをtarget_refへ入力
2. confirmationへRESTOREと入力
3. Run workflow

履歴を強制的に書き換えず、正常状態を新しい復元コミットとしてmainへ反映します。
